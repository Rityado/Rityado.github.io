📱 EIGHT 顧客向けアプリ UI設計書

プロジェクト名: パーソナルジムCRMデジタル化プロジェクト

ドキュメント種別: UI設計書（顧客向けアプリ）

作成日: 2025年11月7日

バージョン: 3.0（全16画面対応・記録機能拡充版）

対象プラットフォーム: Zoho Creator（モバイルWeb/iOS/Android）

📋 1. ドキュメント概要

1.1 目的

本ドキュメントは、パーソナルジム「EIGHT」の顧客向けモバイルアプリケーションのUI設計を定義します。顧客が直感的に予約・記録・レポート閲覧を行える使いやすいインターフェースを実現することを目的とします。

1.2 対象読者

  • プロジェクトオーナー（佐伯様）
  • システム構築・実装者（渡久地様）
  • 相談役（齋藤様）
  • 開発・デザインチーム

1.3 アプリの主要機能

顧客向けアプリの4つのコア機能：
  1. 予約機能 - トレーニング・整体の予約登録・変更・キャンセル（カレンダーベース）
  2. 記録機能 - 体調記録・食事記録・個人トレーニング記録・カウンセリングシート
  3. レポート機能 - トレーニング履歴・整体履歴・Inbody・食事履歴・体調変化の閲覧
  4. レポート一覧（ハブ） - 全レポート・記録機能へのアクセスと主要指標のサマリー表示

📐 2. UI設計の基本方針

2.1 デザインコンセプト

項目 方針
シンプル・直感的 ジムや整体利用後でも疲れた状態で操作できる、シンプルで分かりやすいUI
モバイルファースト スマートフォンでの操作を最優先に設計（片手操作を考慮）
視認性の高さ 大きめのタップエリア、読みやすいフォントサイズ（最小14px）
一貫性 全画面で統一されたナビゲーション・色使い・ボタン配置
フィードバックの明確化 操作に対する成功・エラーメッセージを明確に表示

2.2 カラースキーム（EIGHTロゴベース）

用途 カラー 説明
プライマリ #000000 メインアクション、ヘッダー、ボタン
セカンダリ #333333 ナビゲーション、サブ要素
アクセント（濃） #666666 選択状態、ホバー効果
アクセント（薄） #999999 無効状態、補足テキスト
成功 #2ecc71 成功メッセージ、完了状態
警告 #f39c12 注意喚起、確認メッセージ
危険 #e74c3c キャンセル、削除アクション

🎨 デザインコンセプト： EIGHTロゴのモノトーンを基調とした、シャープで洗練された印象。黒を効果的に使用し、プロフェッショナルなフィットネスブランドを表現します。

🔐 8. セキュリティとプライバシー

8.1 認証・認可

  • ログイン認証：メールアドレス・パスワードによる認証
  • セッション管理：30日間有効（自動ログアウトなし）
  • パスワードポリシー：最小8文字、英数字混在推奨
  • アクセス制御：顧客は自分のデータのみ閲覧・編集可能

8.2 データ保護

  • 通信暗号化：HTTPS通信のみ
  • 個人情報：CRM側で暗号化保存
  • ログ管理：アクセスログ・操作ログをCRMで記録

8.3 プライバシー配慮

  • 体調記録・Inbodyデータなどの機微情報は本人のみ閲覧可能
  • トレーニング・整体履歴は担当トレーナーのみ閲覧可能
  • 利用規約・プライバシーポリシーをアプリ内で常時確認可能
